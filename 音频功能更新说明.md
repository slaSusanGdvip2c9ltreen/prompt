# Sora提示词生成器 - 音频功能更新说明

## 更新日期
2025年11月7日

## 更新目标
1. 提示词主体使用中文，但为了适配Sora，所有口播内容必须使用英文
2. 增强对参考视频口播、BGM的解析和复刻能力
3. 能够处理精美音乐风格设计的视频（极少数情况）
4. 能够处理幽默段子类有口播内容的视频

## 主要更新内容

### 1. 核心概念扩展
- 新增【音频基因】概念：视频的音频元素构成（口播内容、BGM风格、音效设计）

### 2. 音频分析框架（第五步）
在导演级解构分析阶段，新增"音频基因解析"环节，包括：

#### 2.1 口播内容分析
- 是否有人声口播
- 说话人身份（旁白、角色独白、对话等）
- 说话内容的核心信息和情绪基调
- 说话风格（幽默段子、严肃解说、温情讲述等）
- 口播节奏（快速、舒缓、抑扬顿挫）
- **【关键】所有口播内容必须翻译为英文并标注**

#### 2.2 BGM和音乐风格分析
- 音乐类型（流行、古典、电子、民谣、说唱等）
- 音乐情绪（欢快、悲伤、紧张、温馨、搞笑等）
- 音乐节奏（快节奏、慢节奏、节奏变化）
- 音乐与画面的配合方式（卡点、烘托氛围、对比反差）
- 是否有标志性音乐元素（特殊乐器、旋律特点）

#### 2.3 音效设计分析
- 特殊音效识别
- 音效与动作的配合时机
- 音效如何增强画面的冲击力或情绪

#### 2.4 音频分类判断
将视频音频分为四种类型：
1. **纯环境音**：只有自然的环境声音，无BGM无口播
2. **BGM配乐型**：有精心设计的背景音乐
3. **口播解说型**：有人声旁白或解说
4. **综合型**：口播+BGM+音效的复杂组合

### 3. 动态音频指令系统
根据音频类型，提供四种音频指令模板：

#### 3.1 纯环境音类型
```
视频只包含自然的环境音效，绝对没有任何背景音乐或BGM。
```

#### 3.2 BGM配乐型
```
视频配有[音乐类型]风格的背景音乐，[音乐情绪]的旋律，[节奏描述]，[音乐与画面的配合方式]。
```
示例：
```
视频配有轻快的电子音乐作为背景，欢快明亮的旋律，节奏感强，音乐在关键动作处精准卡点。
```

#### 3.3 口播解说型（关键：口播必须用英文）
```
视频包含[说话人身份]的口播，内容为："[口播内容的英文翻译]"，语调[口播风格描述]。[如有BGM或环境音，继续描述]
```
示例：
```
视频包含男性旁白的口播，内容为："This little puppy has no idea how dangerous he's being, but the big dog knows exactly what's up"，语调幽默诙谐带有温情。背景有轻柔的环境音。
```

#### 3.4 综合型
```
视频包含[口播描述，英文内容]，配有[BGM描述]，以及[关键音效描述]。
```

### 4. 口播内容翻译规则
- 所有口播内容必须从原语言翻译为英文
- 保持原意和情绪基调
- 如果是幽默段子，要保留笑点和语言风格
- 如果是解说，要保持专业性和清晰度
- 翻译后用英文双引号括起来

### 5. 输出格式增强
在"爆款基因"部分增加"音频基因解析"输出：

```
音频基因解析：
  音频类型：[纯环境音/BGM配乐型/口播解说型/综合型]
  【如有口播】
    口播身份：[旁白/角色独白/对话等]
    口播风格：[幽默段子/严肃解说/温情讲述等]
    口播内容概要：[核心信息]
    口播英文示例：[关键句子的英文版本]
  【如有BGM】
    音乐类型：[流行/古典/电子等]
    音乐情绪：[欢快/悲伤/紧张等]
    音乐节奏：[快节奏/慢节奏等]
    音乐与画面配合：[卡点/烘托氛围等]
  【如有特殊音效】
    关键音效：[描述主要音效及其作用]
```

### 6. 新增示例
为三种音频类型分别提供了完整示例：
1. **示例1**：纯环境音类型（保持原示例）
2. **示例1-B**：带有口播的幽默段子类型（展示口播翻译为英文）
3. **示例1-C**：带有精美BGM的类型（展示BGM详细描述）

### 7. 工作流程更新
在"开始工作"的特别提醒中，新增【音频处理关键要求】：
- 必须完整分析参考视频的音频元素（口播、BGM、音效）
- 所有口播内容必须翻译为英文并保持原有风格
- 根据音频类型选择正确的音频指令模板
- 对于精美的BGM设计，要详细描述音乐类型、情绪、节奏和与画面的配合
- 对于幽默段子类口播，要保留笑点和语言风格的英文翻译

## 使用场景示例

### 场景1：纯环境音视频
参考视频只有自然声音，无BGM无口播
→ 使用纯环境音模板：`视频只包含自然的环境音效，绝对没有任何背景音乐或BGM。`

### 场景2：幽默段子类视频
参考视频有中文口播："这小狗完全不知道自己有多危险，但大狗心里门儿清。"
→ 翻译为英文：`视频包含男性旁白的口播，内容为："This little puppy has no idea how dangerous he's being, but the big dog knows exactly what's up"，语调幽默诙谐带有温情。背景有轻柔的环境音。`

### 场景3：精美BGM配乐视频
参考视频有温馨的钢琴配乐，在关键时刻音量变化
→ 详细描述：`视频配有温馨的钢琴旋律作为背景音乐，缓慢舒缓的节奏，轻柔的弦乐伴奏，音乐在成年犬站起来的时刻（5秒处）音量微微增强，烘托出守护的温情氛围。`

### 场景4：综合型视频
参考视频同时有角色对话、紧张BGM和音效
→ 综合描述：`视频包含角色对话，内容为："Watch out! That's too close!"，语调紧张急促。配有紧张的弦乐背景音乐，快节奏，在冲突升级时音量增大。同时有狗叫声和脚步声等环境音效。`

## 关键注意事项
1. ⚠️ **提示词主体仍使用中文**，只有口播内容使用英文
2. ⚠️ **口播翻译要保持风格**，幽默段子要保留笑点，不能生硬直译
3. ⚠️ **BGM描述要详细**，不仅要说有音乐，还要描述类型、情绪、节奏、与画面的配合
4. ⚠️ **音频分析是强制步骤**，不可跳过
5. ⚠️ **每条提示词都必须包含音频指令**，根据音频类型选择对应模板
